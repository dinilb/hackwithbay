How each layer is used
Commerce layer

Butterbase

  • Postgres — products, suppliers, offers, orders, payment_methods. The Mercato catalog lives here.
  • Serverless functions — search, shop-agent, place-order deployed as managed endpoints.
  • Auth + billing — user login, Free/Pro plans, card-on-file storage (test mode).
  • AI gateway — Claude ranks the top 4 dealer offers with a one-line reason each.
  • Frontend — Vite + React dashboard deployed to Butterbase.
Relationship layer

Neo4j

  • Property graph — Products, Suppliers, Offers, and Orders as nodes; SOLD_BY, FULFILLS, TRUSTS relationships.
  • Graph traversal — Cypher queries walk supplier chains and rank dealers by reliability × fulfillment, not flat SQL joins.
  • Provenance paths — every order traces back: user → product → supplier → offer → confirmation. Click a receipt, see the path.
  • Live canvas — the dashboard graph visualizes supplier nodes pulsing during search, then lights the buy path green.
Pipeline layer

RocketRide Cloud

  • shop-agent pipeline — deployed to cloud.rocketride.ai, not localhost. Managed production endpoint.
  • Steps — receive NL query → call Mercato search → LLM ranks top 4 offers → stream narration events back to the UI.
  • Observability — every step (search, rank, present) is visible in the RocketRide dashboard for debugging and demo.
  • Integration — Butterbase shop-agent function calls the RocketRide endpoint; the web app and MCP both hit the same pipeline.
